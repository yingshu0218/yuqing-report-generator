# 舆情分析报告生成器 (Yuqing Report Generator)

> 专业舆情分析报告生成 Skill for WorkBuddy / Claw

## 简介

专为**舆情监测与分析**场景设计的 AI Skill，充当资深舆情分析师。

输入话题关键词，自动检索权威媒体信息源（P1 官方媒体 / P2 主流商业媒体），输出一篇**符合官方媒体风格的舆情分析报告** + **信息来源分析文档**。

若未指定话题，自动列出近期具有舆情讨论度的热门新闻标题，供用户选择后生成报告。

---

## 功能特性

- ✅ **自动检索**：5 个维度检索权威媒体信源（新华社、人民日报、澎湃、财新等）
- ✅ **信息核验**：关键事实要求 2 个以上独立来源交叉核验
- ✅ **标准格式**：事件概述 → 舆情焦点（一是/二是/三是）→ 各方观点 → 建议
- ✅ **来源追踪**：所有引用数据/观点标注媒体来源
- ✅ **双文档输出**：舆情报告 + 信息来源分析文档（含可信度评级）
- ✅ **热门话题发现**：无指定话题时自动发现近7天热门舆情供选择

## 输出文档

### 📄 舆情分析报告（`舆情报告_[话题]_[日期].md`）

```
# [话题标题]

## 事件概述
据[媒体名]报道...（2-4段，客观描述核心事件）

## 舆情焦点
一是[焦点]。[分析+引用媒体观点/网友声音]
二是[焦点]。[分析]
三是[焦点]。[分析]

## 各方观点
**官方/支持方：** ...
**质疑方/网友：** ...

## 数据与影响
[数据+影响范围，均标注来源]

## 建议
[2-4条针对相关方的专业建议]
```

### 📊 信息来源分析（`信息来源分析_[话题]_[日期].md`）

- 核心来源清单（来源名称 / 类型 / 日期 / URL / 摘要 / ⭐可信度评级）
- 关键数据核查表（✅已核实 / ⚠️单源 / ❌存疑）
- 信息缺口说明
- 信源构成整体评估

## 安装方式

### 方式一：通过 WorkBuddy 安装（推荐）

```bash
git clone https://github.com/yingshu0218/yuqing-report-generator.git ~/.workbuddy/skills/yuqing-report-generator
```

### 方式二：手动复制

将 `SKILL.md` 复制到 skills 目录：

```
~/.workbuddy/skills/yuqing-report-generator/SKILL.md
```

### 方式三：通过 Claw 安装

在 Claw 平台添加 skill，指向本仓库 GitHub 地址。

## 使用示例

**指定话题：**
```
生成一篇关于"房东税"的舆情报告
```

**不指定话题：**
```
帮我生成一篇舆情报告
```
→ Skill 自动列出5-8个近期热门舆情话题，选择后生成报告。

## 信息来源规范

| 优先级 | 来源类型 | 示例 |
|--------|----------|------|
| P1 最高 | 官方媒体、政府官网 | 人民日报、新华社、央视新闻 |
| P1 最高 | 主流商业媒体 | 澎湃新闻、财新、第一财经 |
| P2 | 媒体社交平台官号 | 人民日报微博、央视新闻微信 |
| P2 | 专业垂直媒体 | 界面新闻、36氪、虎嗅 |
| ❌ 禁止 | 匿名信源、无署名自媒体 | — |

## 报告风格参考

本 Skill 输出风格参照《舆情分析报告》文档合集，典型段落如下：

> 近日，[事件简述]，引发广泛关注。据[媒体名]报道，[核心信息]。相关话题在社交平台迅速传播。
>
> 舆情焦点集中在三条链路：一是[焦点]。[分析]。二是[焦点]。[分析]。三是[焦点]。[分析]。
>
> 建议：一是[建议]。二是[建议]。
